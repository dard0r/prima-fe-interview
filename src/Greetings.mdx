## Introduction
It was fun tackling this challenge because it allowed me to dust off my React knowledge, which I acquired independently out of curiosity and to complete my skillset, but rarely applied in an **"Enterprise"** context.  
In particular, it allowed me to understand how React fits into an environment where reusability, ease of use, and extensibility are fundamental aspects.

## Storybook
Storybook was my adventure companion because it allowed me to quickly test all scenarios.
I think that, today, it is an essential tool in the development of a Design System, because it provides:
- To the contributor, a tool that allows testing what has been developed (and comparing it to the prototypes), the different scenarios, and verifying that no regressions have been introduced;
- To the user, always up-to-date documentation on how to use the components in their application.
- To the designer, who can verify and validate what has been developed.

## What else would I have done?
For completeness, I leave below a list of other aspects that I have omitted but that I consider fundamental in the development of a component for a Design System:
- Documentation and code snippets
- Automated tests (units, integrations, and visual)
- Design tokens
